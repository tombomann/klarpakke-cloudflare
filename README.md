# klarpakke-cloudflare

Cloudflare-native proof-of-concept for the Klarpakke trading-education platform.

Built with **Next.js 16**, **React 19**, **OpenNext**, and **Cloudflare Workers**.

## Staging URL

Deployed to Cloudflare Workers:

- **https://klarpakke-cloudflare.autoglassnorge.workers.dev**
- Health endpoint: `/api/health`

## Tech stack

- Next.js 16.2.9
- React 19
- TypeScript 5.9
- @opennextjs/cloudflare 1.19.11
- Wrangler 4.103

## Scripts

```bash
npm install
npm run build          # OpenNext build → .open-next/worker.js
npm run preview        # Build + local wrangler preview server
npm run deploy         # Build + deploy to Cloudflare Workers
npm run dev            # Standard Next.js dev server
npm run type-check     # TypeScript check
```

## Notes

- `open-next.config.ts` sets `buildCommand: "npx next build"` so that `opennextjs-cloudflare build` does not recursively invoke `npm run build`.
- `wrangler.jsonc` is intentionally minimal: no D1, Queues, or Durable Objects bindings for this Hello World spike.
- API routes use `runtime = "nodejs"` because the edge runtime caused a `Cannot read properties of undefined (reading 'default')` loader error in this OpenNext version.
- Static assets are served via the `env.ASSETS` binding generated by OpenNext.
